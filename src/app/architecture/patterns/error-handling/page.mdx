# 에러 처리 패턴

## 1. 왜 필요한가

일관된 에러 처리가 없으면 디버깅이 어렵고 사용자 경험이 나빠집니다.

## 2. 핵심 개념 요약

- **Global Exception Handler** (Spring: `@ControllerAdvice`)
- **에러 응답 표준화** (RFC 7807 Problem Details)
- **에러 코드 체계** — 도메인별 코드 정의

## 3. 코드 예시

```java
// Spring 전역 예외 처리
@RestControllerAdvice
public class GlobalExceptionHandler {

  @ExceptionHandler(EntityNotFoundException.class)
  public ResponseEntity<ProblemDetail> handleNotFound(EntityNotFoundException e) {
    ProblemDetail pd = ProblemDetail.forStatusAndDetail(HttpStatus.NOT_FOUND, e.getMessage());
    return ResponseEntity.status(HttpStatus.NOT_FOUND).body(pd);
  }
}
```

## 4. 실무 팁

- 예외 메시지에 민감 정보(스택 트레이스, DB 쿼리) 노출 금지
- 로그에는 상세 정보, 응답에는 추상화된 메시지
