# 헥사고날 아키텍처 (Hexagonal / Ports & Adapters)

## 1. 왜 필요한가

외부 시스템(DB, 외부 API)과 비즈니스 로직을 완전히 분리합니다.

## 2. 핵심 개념 요약

- **Port (포트)**: 비즈니스 로직이 노출하는 인터페이스
- **Adapter (어댑터)**: 포트의 구체적인 구현체 (JPA, REST Client 등)
- **Inbound Port**: 외부 → 도메인 (UseCase 인터페이스)
- **Outbound Port**: 도메인 → 외부 (Repository 인터페이스)

## 3. 코드 예시

```java
// Outbound Port (인터페이스)
public interface ArticleRepository {
  Article save(Article article);
  Optional<Article> findById(Long id);
}

// Adapter (JPA 구현체)
@Repository
public class JpaArticleRepositoryAdapter implements ArticleRepository {
  // JPA 구현
}
```

## 4. 실무 팁

- 도메인 계층은 Spring, JPA 등 프레임워크 의존성 없이 순수 Java로 유지
- 테스트 시 포트 인터페이스 기준으로 Mock 생성
