# JPA / 데이터 접근

## 1. 왜 필요한가

직접 SQL 작성 없이 객체 지향적으로 데이터베이스를 다룰 수 있습니다.

## 2. 핵심 개념 요약

- **영속성 컨텍스트**: 1차 캐시, 변경 감지, 쓰기 지연
- **즉시 로딩 vs 지연 로딩**: 기본은 LAZY, 필요할 때만 EAGER
- **N+1 문제**: fetch join 또는 @EntityGraph로 해결
- **QueryDSL**: 타입 안전한 동적 쿼리

## 3. 코드 예시

```java
// N+1 해결 - fetch join
@Query("SELECT a FROM Article a JOIN FETCH a.author WHERE a.id = :id")
Optional<Article> findByIdWithAuthor(@Param("id") Long id);
```

## 4. 실무 팁

- `@Transactional(readOnly = true)` 조회 메서드에 기본 적용
- 양방향 연관관계 mappedBy 주의
