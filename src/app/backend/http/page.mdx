# HTTP / REST API 설계

## 1. 왜 필요한가

잘 설계된 API는 클라이언트 개발을 쉽게 만들고 유지보수를 단순하게 합니다.

## 2. 핵심 개념 요약

- **HTTP 메서드 의미론**: GET(조회), POST(생성), PUT(전체 수정), PATCH(부분 수정), DELETE(삭제)
- **상태 코드**: 2xx(성공), 4xx(클라이언트 오류), 5xx(서버 오류)
- **RESTful 리소스 설계**: 명사 중심, 계층 구조
- **멱등성(Idempotency)**: GET, PUT, DELETE는 멱등

## 3. 코드 예시

```
# RESTful 엔드포인트 설계
GET    /api/v1/articles          # 목록 조회
POST   /api/v1/articles          # 생성
GET    /api/v1/articles/{id}     # 단건 조회
PUT    /api/v1/articles/{id}     # 전체 수정
PATCH  /api/v1/articles/{id}     # 부분 수정
DELETE /api/v1/articles/{id}     # 삭제
```

## 4. 실무 팁

- API 버전은 URL(`/v1/`)에 포함하거나 헤더(`Accept: application/vnd.api+json;version=1`)로 처리
- 에러 응답은 일관된 형식 유지 (RFC 7807 Problem Details)
