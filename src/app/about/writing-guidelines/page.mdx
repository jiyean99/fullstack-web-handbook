# 작성 규칙 (Writing Guidelines)

## 파일 네이밍

- 상위 주제: `page.mdx` (디렉토리 이름이 곧 주제)
- 세부 주제: `rest-api-design.mdx` 처럼 **명사/개념** 중심 kebab-case

## 문서 템플릿

모든 MDX 파일은 아래 구조를 따릅니다:

```mdx
# 제목

## 1. 왜 필요한가
## 2. 핵심 개념 요약
## 3. 코드 예시
## 4. 실무 팁
## 5. 참고 링크
```

## 컴포넌트 원칙

- UI 라이브러리 사용 금지 — 모든 컴포넌트 직접 구현
- styled-components는 Client Component에서만 사용
- 각 컴포넌트에 Storybook 스토리 작성

## 커밋 컨벤션

- `feat`: 새 페이지/컴포넌트 추가
- `docs`: 문서 내용 작성/수정
- `style`: 스타일 변경
- `refactor`: 코드 구조 개선
- `fix`: 버그 수정
