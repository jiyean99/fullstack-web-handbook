# 상태관리 (State Management)

## 1. 왜 필요한가

컴포넌트 트리가 깊어질수록 props drilling이 발생합니다.
상태관리 라이브러리는 전역 상태를 효율적으로 공유하고 관리합니다.

## 2. 핵심 개념 요약

- **로컬 상태**: `useState`, `useReducer` — 단일 컴포넌트 범위
- **서버 상태**: TanStack Query — API 데이터, 캐싱, 동기화
- **클라이언트 전역 상태**: Zustand — UI 상태, 사용자 설정

## 3. 코드 예시

```ts
// Zustand store 예시
import { create } from 'zustand'

interface CounterStore {
  count: number
  increment: () => void
}

const useCounterStore = create<CounterStore>((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
}))
```

## 4. 실무 팁

- 서버 상태와 클라이언트 상태를 명확히 구분하기
- Context API는 성능 이슈 주의 (값 변경 시 모든 구독 컴포넌트 리렌더)
