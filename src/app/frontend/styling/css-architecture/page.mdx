# CSS 아키텍처

## 1. 왜 필요한가

일관된 CSS 구조 없이는 스타일 충돌과 유지보수 어려움이 발생합니다.

## 2. 핵심 개념 요약

- **Design Token** — 색상, 타이포그래피, 스페이싱을 변수로 관리
- **컴포넌트 단위 캡슐화** — 스타일 누출 방지
- **반응형 전략** — Mobile First

## 3. 코드 예시

```ts
// theme.ts - 디자인 토큰 정의
export const theme = {
  colors: {
    primary: '#2563eb',
    text: '#111827',
  },
  spacing: {
    4: '1rem',
    8: '2rem',
  },
} as const
```

## 4. 실무 팁

- `theme.ts`에서만 원시값 정의, 컴포넌트에서 직접 hex 값 사용 금지
- Storybook에서 테마 프리뷰 필수
